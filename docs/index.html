这仓库到底解决什么问题

不是“不会写”，而是“老差半截”

AI 写了，但没写全

focus 补了效果，忘了 `_desc`；event 能跑，namespace 和选项本地化没跟上；decision 改了一半，脚本助手没抽出来。

脚本看着像对，进游戏不生效

很多时候不是逻辑不会写，而是 scope、引用、文件位置、配套资源有一处偏了，最后整段都白干。

日志一炸，不知道先修哪类

报错多的时候最怕乱修。这套东西的核心思路就是先把结构和高频低级错拦下来，再解决真正的玩法逻辑。

核心组成

三样东西一起用，效果才稳

VS Code + CWTools

先把语法、作用域、引用这类编辑期就能发现的问题拦下来，别让 AI 每次都靠猜。

hoi4-modding Skill

让 AI 按真实项目结构干活：先看邻近文件，再按 bundle 生成，最后做审计，不许只改孤立片段。

本地 MCP / CLI

拿来查 mod 根目录、盘点 ID、查缺失 localisation、汇总 error.log，把机械活交给工具。

推荐上手顺序

别一上来就让 AI 写一整套事件链

  1. 先识别 mod 根目录，确认哪个目录才是真正的工作目录。
  2. 先盘点现有结构、命名风格、namespace 和已有 ID。
  3. 再让 AI 生成或修改内容，而且必须按内容包一起动。
  4. 改完马上跑 localisation 审计、ID 审计和日志汇总。
  5. 如果还要做国家、州、地图类重活，直接按“任务公式”那套流程走，不要临时凭感觉编。
